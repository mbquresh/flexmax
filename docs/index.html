02 / THE MISS

The whole product is one moment: what happens when you miss.

Rescheduling a missed task moves a block. It doesn't ask why the block keeps getting missed. Miss the gym forty times and a planner will cheerfully schedule gym #41, identical to #1.

FlexMax counts. When a block has been missed three or more times, the recovery sheet stops being polite about it and names the pattern — then offers the next real slot in your day. No red X, no streak funeral. A question and a way forward.

03 / CHECK-IN

Three taps of honesty, not a binary.

Done/not-done throws away the interesting part. Crushed it, partly, lost focus — the middle option is where the signal lives, and it's the one every habit tracker refuses to offer.

04 / HONEST STATE

What's real, and what isn't.

A portfolio page that claims everything works is a portfolio page nobody believes. So, precisely:

Shipped and working

  • Conversational AI onboarding → psychology profile
  • Schedule builder: drag, swap, fixed anchors, per-day repeats
  • Check-ins, miss reflections, recovery + reschedule
  • Push notifications, per-user local timezone
  • Rate-limited AI endpoints, key server-side only
  • Miss counts read back into recovery prompts

Not built yet

  • The full behavioral learning layer — swaps, ratings and removal reasons are captured but not yet read back
  • Payments / paywall — deliberately deferred
  • Screen Time shielding — pending an Apple entitlement
  • Widgets, Apple Watch, integrations
  • External testers. Users. Revenue.

The app is roughly half of what it should be, and the magic is in the other half. That's the honest number.

05 / BUILD

How it's put together.

Mobile
Expo / React Native — Expo Router, Zustand, Reanimated gestures for drag-to-swap
Backend
Supabase — Postgres with RLS, auth, 11 migrations, edge functions
AI
Claude API, called only from edge functions — onboarding chat, profile extraction, schedule tips, missed-block recovery. No key ever ships in the client.
Notify
Expo Notifications driven by Supabase cron, resolved against each user's local timezone
Design
One token file. No hex values or magic numbers in any StyleSheet — everything imports from theme.ts
